第一章 流量思维：为什么还要做网站

1.1 一切从流量思维开始

做网站，本质上是一种流量思维的体现。

无论你是在抖音发视频、在小红书写笔记，还是在知乎写文章，目的都是为了获取流量。流量在哪里？流量在搜索引擎里，在推荐算法里，在社交媒体的转发里。

而网站，是承接流量的最佳容器。

当你有了网站，你就拥有了： - 可控的流量：不依赖平台算法，不担心账号被封 - 可控的用户：用户数据在自己手里 - 可控的收入：Adsense广告、联盟导购、直接收费，多种变现方式

流量思维的关键是：先获取流量，再考虑变现。而不是反过来，先想怎么赚钱，再去找流量。

1.2 出海做网站的优势

为什么要做英文网站（出海）？核心原因是ECPM（每千次展示收入）的巨大差距：

地区 ECPM（美元）
欧美发达国家 3-10美元
东南亚 0.5-1美元
中国 0.1-0.3美元

做一个面向美国用户的网站，ECPM可能是国内网站的30-100倍。这意味着同样的流量，收入差了30-100倍。

所以哥飞反复强调：做英文站，不是只做英文，可以加上多语言支持，但首要目标是英文世界。

1.3 Adsense：最简单的大众变现方式

变现有多种方式： - 向用户收费：做工具收费、社群收费、课程收费——最赚钱但最复杂 - 联盟导购：Amazon Associates等——需要一定流量基础 - 广告变现（Adsense）：最简单，新手友好

Adsense最简单的模式是：做一个网站 → 放上广告代码 → 躺赚10年。

哥飞举过一个例子：他在2012年创建了一个域名查询网站"秋玉米"，代码10年没改过，到现在还在持续产生收入。这就是Adsense的魅力——做一次，管十年。

当然，躺赚的前提是：你需要多个网站分散风险。一个网站可能收入逐渐减少，但10个、20个网站叠加，收入就非常可观了。

这就是养网站防老的核心逻辑。

1.4 Adsense收入估算公式

理解Adsense的收入模型，是做网站决策的基础。

基础概念：

  • PV（Page View）：页面访问次数，用户每打开一个页面算1次PV
  • UV（Unique Visitor）：独立访客数，同一用户多次访问只算1次UV
  • ECPM（Effective Cost Per Mille）：每千次广告展示收入，单位是美元
  • 可见率：广告被实际看到才算展示，Scroll广告通常可见率约80%

核心公式推导：

首先理解"每千次展示"的概念： - 1个页面放3个广告位 → 1次PV = 3次广告展示机会 - 1000次展示 ÷ 3个/页 = 333次PV才能产生1000次展示 - 如果ECPM=9美元，即1000次展示收入$9 - 那么：333 PV → $9 → 每111 PV → $3 → 每37 PV → $1

工具站的特点是用户查完一个词会继续查下一个，平均浏览3-5个页面。

按保守的3个页面计算（每来110人有330PV）：

每来110个访客就有330PV，就有10美元收入 每来11个访客就有1美元收入

这是数据好的网站（ECPM接近10美元）。数据差的网站也有2美元ECPM。

标准公式：

月收入(美元) = (月PV × 广告位数 × 可见率 × ECPM) / 1000

简化版（每页3个广告位、80%可见率）：

月收入(美元) ≈ 30 × UV基数 × ECPM(美元) × PV/UV比

公式解释： - 30 = 基准常数（1 UV × 1 ECPM × 1 PV × 1广告位 = $1/天 × 30天） - UV基数 = 日独立访客数（单位：1000） - ECPM = 每千次广告展示收入 - PV/UV = 每访客浏览页面数 - 广告位数 = 每页广告数量（最多3个）

实际计算案例：

一个面向美国用户的查询工具站： - 日UV = 10,000（基数=10） - ECPM = 4美元（欧美发达国家） - PV/UV = 3（查询工具站用户通常浏览3-5页） - 广告位 = 3

月收入 = 30 × 10 × 4 × 3 × 3 = $10,800 ≈ ¥78,000

1.5 关键词ECPM差异

同一关键词在不同国家，ECPM差距巨大。

金融、法律、健康类关键词的ECPM远高于娱乐、通用工具类。

工具站的特点是用户查询意图明确，愿意在页面上停留，广告展示率高，是做Adsense的优质网站类型。

第二章 需求挖掘：找到值得做的关键词

2.1 核心方法：看榜单、看数据、看他人在做什么

需求挖掘的真传一句话：看榜单，看数据，看别人都做了什么。

第一步：用Similarweb看竞品

打开 Similarweb（https://www.similarweb.com/ ），无需注册即可查看网站月访问量、流量来源、搜索关键词等数据。

实操步骤（以挖掘在线工具需求为例）：

  1. 在谷歌搜索"在线工具"，找到排在前列的网站，如 tool.lu
  2. 打开 Similarweb，输入目标域名，地址：https://www.similarweb.com/zh/website/tool.lu/#overview
  3. 首先查看月访问量（如 tool.lu 月访问量170万），判断网站整体规模
  4. 查看用户兴趣 → 其他受访网站，发现 json.cn、bejson.com、cnblogs.com、w3cschool.cn 等同类网站，右键新标签页逐个打开备用
  5. 查看竞争对手和类似网站，同样逐个右键打开备用
  6. 查看流量来源分布，分析直接访问与搜索引擎（自然搜索）的比例，比例高说明SEO价值大
  7. 查看搜索关键词，可以看到排名前几的搜索词（如"时间戳"、"时间戳转换"、"ip地址查询"等），每个词都可以单独做一个工具站

从 tool.lu 可以挖到的关键词包括：时间戳、时间戳转换、IP地址查询、IP查询、IP、时间戳转换器、在线工具、整数分区、视频地址解析、下载地址转换、整数分区计算器、unix时间戳、timestamp、unixtimestamp、在线拆字。每一个都可以注册域名单独做一个工具站。

第二步：用爱站网查百度权重

实操步骤：

  1. 打开爱站网首页 https://www.aizhan.com ，在下拉框选择"百度权重"，输入竞品域名（如 tool.lu），点击"查询"按钮
  2. 打开的地址格式为 https://baidurank.aizhan.com/baidu/tool.lu/
  3. 拉到页面中部，查看PC端和移动端的涨入/跌出关键词列表，了解近期排名上升和下降的词
  4. 继续查看页面下方表格，这里列出了该站点所有有排名的关键词列表，第三列为预估的PC端百度搜索量
  5. 用爱站数据交叉验证Similarweb中挖掘到的关键词，确认搜索量和竞争情况

第三步：用站长之家验证SEO难度（可选）

  1. 打开 https://tool.chinaz.com/kwevaluate ，输入目标关键词
  2. 查看整体优化难度评分：低于86分可以尝试，超过90分竞争激烈建议换词
  3. 重点关注"前50名搜索结果中首页数量"，首页数量越少机会越大

完整工作流：Similarweb看竞品数据 → 爱站网查排名关键词 → 站长工具验难度 → 确定做哪个词

3.1 KDRoi公式：选词的量化工具

哥飞自创的关键词价值量化公式：

KDRoi = (搜索量 × CPC) / 优化难度KD

解读： - KDRoi 越高，这个关键词越值得做 - 搜索量大 × CPC高 ÷ KD低 = 高回报 - 用 Semrush 导出关键词数据，在 Excel 中计算

实操步骤： 1. 在 Semrush 输入目标关键词，设置筛选：搜索量≥600，KD为0-29，CPC>0.1 2. 导出关键词表格（CSV或XLSX） 3. 在 Excel 中保留 Keyword、Volume、KD、CPC 四列 4. 新增一列，输入公式 =B2*D2/C2（即 Volume × CPC / KD） 5. 按新列从高到低排序，得分最高的就是最值得做的词

3.3 小词的CPC价值

容易被忽略的小词，往往有很高的CPC。

关键词"CPC"（Cost Per Click，每次点击成本）代表了广告主愿意为这个关键词付多少钱——CPC越高，意味着商业价值越高，用户越愿意付费。

即使搜索量只有几百的"小词"，如果CPC高达10美元以上，其KDRoi可能远超搜索量上万但CPC只有0.1美元的大词。

核心思路：与其做大词的凤尾，不如做小词的鸡头。小词竞争少，容易拿排名，积累下来收入也很可观。

第五章 Google SEO基础：谷歌如何工作

5.1 谷歌排名三句话

理解谷歌排名机制，记住三句话即可：

第一句：Title和H1等页面元素确保你的网页进入关键词搜索的候选范围。

也就是说，如果你的页面Title和H1里没有某个关键词，谷歌基本不会考虑把你的页面展示给搜索这个词的用户。

第二句：外链帮助你进入谷歌搜索前10位置。

即使你的页面进入了候选范围，还需要足够的外部链接才能在众多候选者中脱颖而出，进入前10名。

第三句：网站体验（停留时间、跳出率等）决定最终排名。

进入前10后，用户行为数据会成为最终排名的决定因素。用户越喜欢你的页面（停留久、不跳出、多翻几页），你的排名就越稳定、越靠前。

5.2 谷歌喜欢的页面特征

谷歌本质上是在帮用户找到最合适的页面。所以，谷歌喜欢的页面 = 用户喜欢的页面。

一、用户体验 1. 加载速度快 - 速度慢不仅影响排名，还拖慢谷歌爬虫抓取 2. 移动端友好 - 现在移动端流量已超过50% 3. 内链结构好 - 把新内容在首页列出，是加快收录屡试不爽的方法

二、互动性（用户行为信号） 4. 用户停留时间长 - 说明内容满足了搜索意图 5. 跳出率低 - 不立即关闭，说明内容相关 6. 平均访问页面多 - 用户愿意继续探索

三、内容质量 7. 原创度高 - 不搬运、不抄袭。谷歌能判断整个网站的内容重复度 8. 内容权威 - 把事情掰开揉碎讲清楚 ≠ 名人撰写 9. 内容实用 - 真正解决问题 10. 关键词密度不下降 - 标题与内容匹配，不要堆砌

四、技术优化 11. 定期更新 - 内容随时间可能过时，谷歌喜欢活跃的网站 12. 工具页面也要有文字 - 用文字描述工具功能 13. 合理书写TD - 每个页面TD不同，体现差异化 14. 合理Headings - H1主题、H2二级、H3三级 15. 图片Alt属性 - 让图片进入Google图片搜索

5.3 停留时间与排名

停留时间（dwell time）是重要的排名信号。

用户点击你的页面后，如果很快返回搜索结果并点击其他网站，谷歌会认为你的页面没有满足用户需求，排名下降。

反之，如果用户在你的页面停留很久，说明内容确实有用，排名上升。

第七章 技术SEO：元标签、结构化数据与技术细节

7.1 Canonical标签详解：规范化URL的核心指南

Canonical标签（规范化标签）用于告诉搜索引擎，当前页面的唯一规范网址是哪个，解决重复URL导致的权重分散问题。

7.1.1 为什么需要Canonical

同一个网页在以下情况下会产生多个URL：

（1）www vs 裸域名同时可访问

例如 www.example.com 和 example.com 都能打开同一个页面，加上HTTPS后又多一个变体。

（2）目录默认文件

如 /about/、/about/index.html、/about/index.php 都能访问同一个页面。

（3）跟踪参数

用户从不同渠道分享链接，带有不同的跟踪参数： - 张三分享：/shequn/?ref=zs - 李四分享：/shequn/?via=ls - Reddit引流：/shequn/?r=reddit

以上这些URL返回的内容完全相同，但在搜索引擎眼里是完全不同的"重复网页"。

7.1.2 www与裸域名的处理原则

最佳做法：选定一个作为主域名，另一个通过301重定向过去。

# Nginx 配置示例：将 www 重定向到裸域名
server {
    listen 80;
    server_name www.example.com;
    return 301 $scheme://example.com$request_uri;
}

特殊情况（大网站防止Cookie泄漏）： 有些大型网站会将 www 版作为主域名，把裸域名用301跳转到 www，防止Cookie跨域泄漏。哥飞建议普通网站优先选择裸域名作为主域名。

7.1.3 带参数的URL处理

外链带上跟踪参数（如 ?r=reddit、?ref=zs）是常态。不设置Canonical，这些外链的权重就分散到各个带参数的URL上；正确设置后，所有外链权重汇聚到规范URL。

<!-- 无论用户从哪个带参数的URL进来，规范URL始终是这一个 -->
<link rel="canonical" href="https://example.com/shequn/">

7.1.4 多语言网站的Canonical写法

多语言网站（子目录结构）每个语言版本的Canonical必须指向自己的真实URL：

<!-- 中文页面（默认语言）的Canonical -->
<link rel="canonical" href="https://example.com/shequn/">

<!-- 英文页面的Canonical -->
<link rel="canonical" href="https://example.com/en/shequn/">

<!-- 日语页面的Canonical -->
<link rel="canonical" href="https://example.com/ja/shequn/">

重要：不同语言页面的Canonical值必须不同，各自指向自己的URL。绝对不能所有语言都写死成同一个URL。

7.1.5 正确写法总览

<!-- 首页规范 -->
<link rel="canonical" href="https://example.com/">

<!-- 内页规范 -->
<link rel="canonical" href="https://example.com/china-phone-generator.html">

7.1.6 失败案例

案例一：全站所有页面Canonical都写为首页

有些开发者对Canonical不理解，在所有页面的<head>里都写：

<link rel="canonical" href="https://example.com/">

这会导致搜索引擎认为所有页面都是首页的副本，内页的权重全部流失到首页，整站排名急速下降。

案例二：多语言页面Canonical写错

英文页面错误地写成了中文页面的URL：

<!-- 英文页面错误写法 -->
<link rel="canonical" href="https://example.com/zh/about.html">

搜索引擎会收到混乱信号，不知道这个英文页面究竟对应哪个URL，轻则不收录，重则降权。

案例三：忘记写Canonical导致权重分散

张三在Reddit上给 /shequn/?r=reddit 做外链，李四在Product Hunt上给 /shequn/?r=ph 做外链。如果不设置Canonical，这两个外链的权重分别被谷歌计入两个不同URL，无法叠加，导致排名效果大打折扣。

7.2 Nofollow标签

nofollow 链接的演变：

最初 nofollow 是用来告诉爬虫"不要继续爬这个链接"。但后来，谷歌发现用户自发传播的链接往往是nofollow的，而交易链接才是dofollow的。于是修改算法，现在 nofollow 对权重提升也有作用。

正确理解： - nofollow 外链也有价值，不要嫌弃 - 追求自然的外链组合

nofollow的使用场景： - 用户生成内容中的外链 - 付费/赞助链接 - 不受信任的链接

<a href="外链URL" rel="noopener noreferrer nofollow">锚文本</a>

7.3 robots meta标签

控制单个页面的抓取行为：

<!-- 禁止抓取但允许索引 -->
<meta name="robots" content="noindex, follow">

<!-- 禁止抓取也禁止索引 -->
<meta name="robots" content="noindex, nofollow">

使用场景： - 感谢页面、登录页面等不需要搜索流量的页面 - 内部管理页面

7.4 robots.txt 多语言网站写法与常见错误

多语言网站用子目录形式时，robots.txt 需要逐个语言版本设置禁止规则。

7.4.1 正确写法

User-Agent: *
Allow: /
Disallow: /people/
Disallow: /ja/people/
Disallow: /fr/people/
Disallow: /ko/people/
Disallow: /zh/people/

核心原则：有多少种语言就要写多少行。

7.4.2 失败案例

哥飞在实际项目中就踩过这个坑：网站默认语言的 /people/ 目录正确禁止抓取了，但日语 /ja/people/、韩语 /ko/people/ 等多语言版本仍然被谷歌抓取到了。

原因是 robots.txt 只写了 Disallow: /people/，这只对默认语言生效，对子目录 /ja/、/ko/ 下的路径无效。

7.4.3 通配符陷阱

有人为了省事，会这样写：

User-Agent: *
Allow: /
Disallow: /*/people/

哥飞特别提醒：不要用通配符偷懒，这会导致不可预知的问题。例如你有一个页面 /abc/def/people/ 也会被这个规则覆盖，而它实际上是你希望被抓取的页面。

推荐方式：手动为每种语言写一行禁止规则，并定期检查更新——每当网站新增语言支持时，记得在这里同步增加一行。

技术补充：如果你用的是 Next.js 框架，在 App Router 下可以用 robots.ts 动态生成 robots.txt，里面可以写逻辑自动处理多语言路径。 参考：https://nextjs.org/docs/app/api-reference/file-conventions/metadata/robots

常用结构化数据类型：

类型 适用场景 SEO效果

哥飞建议用子目录而非子域名： - example.com/en/（推荐） - en.example.com（不推荐）

7.8 Viewport Meta标签：移动端适配

Viewport元标签用于配置网页在不同大小屏幕下的缩放和显示方式，是移动端适配的基础。

标准写法：

<meta name="viewport" content="width=device-width, initial-scale=1">

参数含义： - width=device-width：使页面宽度与设备屏幕宽度匹配（以设备独立像素为单位） - initial-scale=1：建立CSS像素与设备独立像素1:1的关系，并考虑屏幕方向

为什么重要： - 谷歌明确表示移动端友好占排名因素的5%权重 - 未设置Viewport的页面在移动端会被强制缩放，用户体验极差 - 移动端友好是现代网站的必备条件

不设置Viewport的效果：页面会被缩放得很小，用户必须放大才能操作，这会直接导致高跳出率和低排名。

第八章 外链建设：nofollow也有权重

8.1 外链的本质

外链是其他网站给你的"投票"。越多高权重网站链接你，谷歌认为你越可信。

外链类型（按质量从高到低）： 1. 政府/教育机构链接（.gov、.edu）—— 极难获得 2. 新闻媒体报道 3. 高权重行业网站 4. 论坛/社区自然提及 5. 社交媒体分享

8.2 nofollow也有权重

重要发现：谷歌已经修改算法，nofollow链接对权重提升也有作用。

原因：用户自发传播的链接通常是nofollow的，而交易链接（花钱买的）是dofollow的。谷歌意识到，如果只看dofollow，会被大量人工操控。所以现在nofollow外链也被计入权重计算。

实际影响：不要只追求dofollow外链，nofollow外链同样有价值，有就要。

8.3 外链建设方法

免费外链渠道： - Reddit/HackerNews 分享产品 - 导航站提交（Tool站集合网站） - Product Hunt 发布 - 社群分享（独立开发者社区）

第九章 内容策略：AI生成高质量内容

9.1 谷歌对AI内容的态度

核心立场：谷歌奖励高质量内容，无论内容是如何生成的。

2023年2月谷歌官方明确表示："奖励高质量的内容，无论它是如何生产的。"

关键点： - 谷歌不看内容是人工写还是AI生成 - 谷歌只看内容是否有用、可靠、以人为本 - 不管生成方式，只看内容质量

9.2 AI内容的常见错误

错误做法：凭空捏造

直接让GPT"帮我写一篇关于xxx的文章"，出来的内容通常是泛泛而谈、缺乏深度的垃圾内容。

特征： - 内容空洞，没有实际价值 - 数据可能胡编乱造 - 用户一眼能看出是AI写的（"首先...其次...最后..."） - 跳出率高，排名反而下降

9.3 正确方法：提供参考资料

核心原则：不要让AI凭空创作，而是提供参考资料让它按格式输出。

操作步骤： 1. 找到一个做得好的参考页面 2. 把参考页面发给GPT，让它学习格式和内容结构 3. 提供你的目标关键词 4. 要求GPT"按照参考页面的格式，生成关于[关键词]的内容" 5. 人工审核，补充本地化信息

提示词模板：

我有一个范文示例：
[粘贴参考内容]
请学习这个范例的格式和风格。
现在请按照这个格式，生成关于"XXX"的内容，
包含H2标题、段落描述和工具介绍。

9.4 保持内容原创性

避免重复： - 谷歌能判断网站与网站之间的内容重复度 - 不要搬运或改写别人已有的内容 - 每篇文章要有独特价值

AI辅助而非AI替代： - 让GPT帮你生成框架和初稿 - 加入你的专业知识、独特见解 - 补充真实数据、案例 - 人类编辑审核把关

9.5 页面内容质量标准

字数标准：单页单词数不低于600个。

高质量内容特征： - 把一个问题讲透彻，而不是蜻蜓点水 - 提供实用的工具或资源 - 回答用户的真实问题 - 内容格式清晰（标题层级分明）

9.6 内容型AI工具站：AI+SEO的新范式

什么是内容型AI工具站？

这是哥飞原创的概念（2024年3月提出）。传统AI工具站是"用户来、用完走"，没有留下任何内容给搜索引擎抓取。而内容型AI工具站的核心区别在于：用户使用工具产生的内容，会自动出现在网站的公共区域（广场），被Google发现并收录。

网站提供工具给用户使用，用户使用过程中产生的内容会出现在广场上被更多人看到。当然也会被Google看到，进而被抓取被索引。当有人在Google搜索某些相关关键词时，我的这个网站的图片就有可能出现在搜索结果里，从而吸引用户点击打开我的网站。

用户用得越多，产生的内容越多，这些内容又通过搜索引擎带来更多的用户。

这就是内容型AI工具站。

核心案例：Sticker.Show

Sticker.Show（https://sticker.show/）是哥飞团队上线的AI贴纸生成器，是内容型AI工具站的典型实践。

完整飞轮效应：

用户生成贴纸 → 内容展示在广场 → Google收录 → 图片出现在搜索结果
→ 更多用户搜索点击进入 → 更多用户使用 → 更多内容产生 → 更大流量

关键点： - 用户输入提示词，AI生成贴纸，生成的贴纸自动展示在公开页面 - 这些页面被Google图片搜索收录 - 搜索相关贴纸关键词的用户会发现并点击进入网站 - 更多用户使用 → 更多内容 → 更大搜索流量

为什么这个模式有价值？

对比维度 传统工具站 内容型AI工具站
用户离开后 什么也没留下 内容留在广场被收录
SEO内容来源 需要自己人工创作 用户使用中自动生成
流量增长 缓慢，靠内容页积累 有飞轮效应，用户越多流量越多
内容丰富度 受限于建站者精力 由用户需求驱动，无限丰富

本质区别：传统工具站的内容需要站长自己创作，而内容型AI工具站把"创作内容"这件事众包给了用户——用户使用工具的同时，也在帮你生成SEO内容。

商业模式：工具免费，增值服务收费

内容型AI工具站的变现设计非常清晰：

免费层： - 基础AI生成功能 - 生成的内容在广场展示

付费层（增值服务）： - 下载高清大图（普通下载免费，高清需付费） - 去除水印 - 私密生成（不公开到广场） - 订阅会员（无限使用）

当用户不想生成的内容被公开，或者想要下载高清大图时，就可以付费订阅。相当于工具免费，增值服务收费。

这种站是垃圾站吗？

显然不是。

判断标准： - 有真实的工具满足用户的个性化需求 - 有真实用户产生的真实内容供大家消费 - 内容来源于用户实际使用，不是凭空捏造的

这与传统的"采集站"、"AI生成内容站"有本质区别——所有内容都是用户在解决自己问题时产生的副产品，完全以人为本。

适用场景与扩展

这个模式可以套用到大量需求上：

输入类型：文字、图片、视频、网址 输出类型：文章、图片、视频、Web App

可以做的内容型AI工具站： - AI贴纸生成 → 头像生成 → 壁纸生成 - AI图片编辑（换背景、修复、去水印） - AI视频剪辑（字幕生成、视频摘要） - AI文档处理（PDF转Word、OCR识别） - AI配色生成、Logo生成、海报生成

哥飞在做Sticker.Show时已经将其模板化，可以快速迁移到其他类型的AI工具站。一次开发，可以上多个站，每个站覆盖不同的用户需求和关键词。

12.5 收入估算实操

12.6 DateTimeGo完整广告费计算

DateTimeGo.com 是程序化SEO的经典变现案例，以下是完整的广告费计算推导过程。

已知数据： - 月访问量：130万UV - 平均PV/UV：1.8（每个访客平均浏览1.8个页面） - 每月总PV：130万 × 1.8 = 234万PV - 广告位数：6个/页（首页+内容区+底部） - 广告可见率：约80%（滚动广告实际被看到的比例） - ECPM：约3美元（日期时间类工具站）

完整计算推导：

月广告展示次数 = 月PV × 广告位数 × 可见率
               = 234万 × 6 × 80%
               = 234万 × 4.8
               = 约1123万次

月广告费 = 广告展示次数 / 1000 × ECPM
         = 1123万 / 1000 × 3
         = 1123 × 3
         = 约3369美元/月

实际验证：原文计算为"1123万×6×0.8×3/1000 ≈ 3.4万美元/月"，与网站后台数据（月广告费3万美元）基本吻合。

这个案例说明什么： 1. 程序化生成页面能够带来稳定的大流量 2. 工具站PV/UV偏低（1.8），但通过大量广告位（6个）弥补 3. ECPM虽然只有3美元（不如金融类10美元），但流量够大，收入依然可观

13.5 浏览器插件+网站：留存问题的创新解法

网站普遍面临一个难题：留存差，用户用完就走，很难让用户持续回来。

而 custom-cursor.com 这个网站，完美地解决了这个问题。

案例背景

custom-cursor.com 是一个帮助用户设置自定义鼠标光标图案的网站。哥飞在谷歌搜索"Cursor"时发现了它，其配套的浏览器插件周活高达400万。

核心机制：流量循环

这个案例的关键在于网站与浏览器插件的协同：

  1. 用户通过谷歌搜索发现网站
  2. 网站引导用户安装浏览器插件
  3. 用户使用插件时，右下角的"更多游标"按钮会打开官网
  4. 官网提供更多光标样式，吸引用户持续访问

这样就形成了完整的流量循环：网站引导安装插件 → 用户使用插件时打开网站 → 流量再回到网站。

第十四章 避坑指南：真实失败案例解析

14.1 深度复盘：首页内容膨胀导致降权的完整过程

这是一个真实发生的SEO失败案例，发生在哥飞自己的套壳AI站 ChatGPT4o.ai 上。哥飞把这个案例完整复盘出来，希望帮助大家避免踩同样的坑。

背景知识：为什么谷歌爬虫会频繁抓取某些网站？

在分析失败原因之前，先理解一个关键概念：谷歌爬虫的抓取频率不是固定不变的。

用 Google site:指令 查询某个网站时，可以把时间范围设置为"过去一小时"（Past hour），这样可以看到谷歌爬虫最近一小时抓取了哪些页面。从 V2EX 的例子可以看到，1小时内谷歌爬取了大量新帖子——这说明 V2EX 是一个非常活跃的社区，时刻都在产生新内容，谷歌为了不错过，就频繁过来爬取。

那么，如果我们想让谷歌爬虫也频繁来访我们的网站，应该怎么做？

答案很简单：在首页增加一个模块，显示最新的几个内页链接。 这样做的好处是：每次搜索引擎爬虫来访，都能发现有新东西可以抓，于是越来越频繁地来访，收录的页面也越来越多。

ChatGPT4o.ai 就是这么做的——在首页问答模块下方增加了一个"最新问答"模块，显示最新的6条问答链接。这本来是一个正确的做法，但问题就出在某次发版之后。

问题发生的完整过程

某次发版之后，工程师的一个"无心之失"改变了一切："最新问答"列表模块把问题和回答的全文都在首页输出了。

什么意思呢？来看看数据对比：

版本 首页单词数 说明
发版前 约400个单词 正常的首页体量
发版后 两三千个单词 问题和回答全文堆在首页

原本只需要展示问答的标题和链接，结果变成了在首页直接展示完整的问答内容。这意味着首页的主题从"GPT 4o AI问答工具"变成了各种杂七杂八的用户问答内容。

关键词密度下降：问题的核心

关键词密度（Keyword Density）指的是某个核心关键词在页面所有单词中出现的比例。首页的核心关键词是"GPT 4o"或"AI"相关词汇，当首页只有400个单词时，核心关键词出现的比例是合理的。

但当首页膨胀到两三千个单词之后，核心关键词的密度骤降——因为填充进来的是大量用户问答的正文内容，这些内容里核心关键词的出现频率远低于原本首页的刻意优化。

如何判断关键词密度？ 有几种常用方法：

  1. 手动估算：将目标关键词在页面中出现的次数除以页面总单词数（通常认为 1%-3% 为合理区间）
  2. 使用SEO工具：AITDK SEO Extension、Semrush 等工具可以自动计算页面关键词密度
  3. Google 语义分析辅助判断：即使关键词密度数字看起来还行，如果页面内容语义上与关键词不匹配，谷歌也会识别出来

谷歌语义分析惩罚机制

关键词密度只是表面问题，更深层的原因是谷歌的语义分析能力。

谷歌不仅仅统计关键词出现次数，它还会分析页面的整体语义主题。谷歌认为：一个自称是"GPT 4o AI问答站"的网站，首页应该围绕 AI 问答这个主题展开内容。如果首页充斥着大量与 AI 工具无关的用户问答（比如"今天天气怎么样"、"给我讲个笑话"等），谷歌语义分析之后发现：你这个网站首页内容乱七八糟的，跟 GPT 4o 没多大关系了。

于是惩罚机制触发：谷歌直接减少该网站的曝光量，降低排名，不给点击。

后果与教训

虽然后来哥飞紧急下掉了这个模块，把首页恢复到了正常状态，但网站的排名至今还没有完全恢复——被谷歌惩罚后的恢复期往往比下滑期更长。

这个案例的核心教训：

  • 首页要保持主题集中：首页是网站的"脸面"，是告诉谷歌"你这个站是做什么的"最关键的地方。首页内容必须围绕核心关键词展开。
  • 不要在首页堆砌内页内容：内页的内容应该在内页展示，不应该把内页全文直接输出到首页。即使是动态生成的内容，也要控制首页体量。
  • 发版前要做SEO检查：任何涉及首页内容结构的变化，在发版前都应该检查对关键词密度和语义主题的影响。
  • 谷歌语义分析已经相当成熟：不要以为关键词密度降到某个阈值以下才会出问题，语义不匹配本身就足以触发惩罚。

14.2 Canonical标签写错导致降权

常见错误： - 网站所有页面都返回首页的 Canonical URL - Canonical URL 和当前页面 URL 不一致

后果：谷歌不知道哪个是真实页面，权重分散或降权。

正确做法：

<!-- 正确的Canonical -->
<link rel="canonical" href="当前页面完整URL">

14.3 robots.txt多语言写法踩坑

错误：

Disallow: /people/

只会禁止默认语言的 /people/ 目录，/ja/people/、/fr/people/ 仍会被抓取。

正确：

Disallow: /people/
Disallow: /ja/people/
Disallow: /fr/people/
Disallow: /ko/people/

有多少种语言版本，就需要写多少行禁止规则。有比没有好，但写错了比不写更糟糕。

14.4 AI内容"凭空捏造"导致低质量

问题：直接让GPT生成内容，不提供参考资料，导致内容空洞、重复率高。

后果： - 谷歌判断重复率高，不给排名 - 用户跳出率高，排名持续下降 - 网站整体权重受损

正确做法： - 始终提供参考资料 - 加入人工审核 - 补充真实数据和案例

14.5 选错关键词白费功夫

问题：没做关键词调研就开发网站，做了半天发现没有搜索流量。

后果： - 竞争激烈的关键词，新站根本没机会 - 搜索量太低的关键词，做了也没流量 - 时间精力浪费

正确做法： - 先用 KDRoi 公式筛选 - 优先选择 KD<29、Volume>600、CPC 较高的词 - 新站不要硬碰大词，从长尾词切入

第二步：逐页面 SEO 分析

（一）首页：核心入口页

做得好的地方：

  1. 工具入口直接放置：首页最核心区域直接放置工具入口，用户一打开就能看到并开始使用，提高页面停留时间。
  2. 最新作品列表：每次刷新取出最新生成的贴纸，对于搜索引擎来说，这是一个频繁更新的页面，爬虫每次来访都有新内容可抓，从而加快收录速度。
  3. 社会认同营造：最新作品列表不断冒出新的贴纸，给用户感觉"这个网站很火，还有很多人在用，我也要试试"。
  4. 社媒分享标签完善：Open Graph 和 Twitter Card 均已设置，分享到社交媒体时显示效果良好。

需要改进的地方：

  1. Title 太短：只用12个字符，没有充分利用 Title 体现需求关键词（如"Sticker Maker"、"AI Sticker Generator"）。
  2. Description 太短：只用21个字符，建议写到接近160字符限制，包含核心关键词和行动号召。
  3. Canonical 缺失：每个页面都应该设置 Canonical 标签，防止重复内容分散权重。
  4. H 标签层级混乱：首页出现两个 H1，且都没有体现网站的核心关键词。建议：
  5. 只保留一个 H1，格式如"AI Sticker Generator - Create Custom Stickers Online"
  6. "Latest" 改为 "Latest Generated Stickers" 并用 H2
  7. "Upload" 不应用 H1 或 H2，建议用 H3
  8. 图片 Alt 可以更长：生成的 Sticker 图片 Alt 如"Ayna"建议改为"Ayna Sticker"，包含更多关键词信息。
  9. 建议增加 Discover 链接：目前"更多贴纸"通过上拉获取，搜索引擎爬虫无法做上拉操作。建议增加 Discover 页面入口，能够分页展示所有贴纸，让爬虫能爬到全部内容。

（二）贴纸详情页（以 /sticker/29961 为例）

URL 结构：

/sticker/29961-fairy-unicorn

URL 直接用数字 ID，容易被爬虫从1开始遍历。生成贴纸时的提示词如果很长，不建议直接拿来做 URL，可以抽取主要关键词。

Title 优化：当前 Title 可能是"I made an AI sticker of fairy unicorn"，其中"I made an"是分享用途的前缀，实际上可以去掉，改为"AI Sticker of Fairy Unicorn - fairy unicorn Sticker"，这样核心关键词"AI Sticker"和长尾关键词"fairy unicorn"都照顾到了。如果担心分享出去标题不好看，可以单独在 Open Graph 和 Twitter Card 中设置带前缀的标题。

H 标签：H1 建议体现核心关键词，如"AI Stickers of fairy unicorn"，"Similar-ish stickers"可放在 H2。

相似贴纸功能：这是神来之笔，可以有效提升长尾关键词的覆盖。但目前是前端获取数据并渲染的，建议改为后端渲染，这样才能被搜索引擎抓取，从而提升关键词密度和排名。

（三）搜索首页（/search）

"Sticker Search"这个关键词有一定搜索量，专门做一个搜索入口页是正确的做法。但注意： - Title 用了"StickerSearch"连写，用户更可能搜索的是带空格的"Sticker Search"，需要在 Title、Description、H1 等处都兼容有空格的写法。 - 目前只用了一个 H5，极其不合理，应该改成 H1，并增加更多内容和标签。

（四）搜索结果页

搜索结果页是获取大量长尾关键词流量的关键，但目前存在严重问题：

  1. 前端渲染导致内容重复：不管搜索什么关键词，Title 和 Description 都固定了，搜索引擎爬虫看来，不同 URL 的页面 HTML 内容都是重复的，这是 SEO 大忌。
  2. 建议改为后端渲染：这样可以利用真实用户搜索生成的大量长尾关键词页面，每个搜索结果页都有独特的 Title（包含搜索关键词）、H1 和 Description。
  3. H 标签应动态生成：例如 /search?query=unicorn，H1 和 Title 都应该是"160 Best Unicorn Stickers"，其中"160"是搜索结果数量。

第三步：改进优先级排序

根据以上分析，给出改进优先级建议：

P0（立即修复，影响排名）： 1. 修复 Canonical 标签缺失问题 2. 搜索结果页改为后端渲染（SEO 价值最大，但改动量也最大）

P1（本周内修复，影响排名）： 1. Title 和 Description 补充完整 2. 修复 H 标签层级（每个页面只保留一个 H1） 3. 相似贴纸改为后端渲染

P2（近期优化，提升流量）： 1. 增加 Discover 分页页面 2. 搜索首页 H 标签和内容补充 3. 增加结构化数据（FAQ、Product 等）

P3（长期优化）： 1. 增加专题页面（用向量搜索实现智能聚合） 2. 增加搜索关键词列表页面（让爬虫能爬到所有搜索结果页） 3. 多语言支持及 hreflang 配置

案例一：单篇帖子狂赚2000+ Karma

案例二：如何通过Comment赚Karma

核心发现：绝大部分Reddit用户的Comment Karma远高于Post Karma，因为Reddit本质是讨论社区，评论比发帖更活跃。

实操方法： 1. 在自己的帖子下积极回复，越早的评论位置越容易获得upvotes 2. 在别人的热帖里找"热坑位"和"早坑位"——找已有很多upvotes的评论，在其下方进一步回复蹭曝光 3. 培养网感，能发掘可能会大火的帖子，第一时间去评论
